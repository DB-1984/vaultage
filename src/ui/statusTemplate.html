01_Auth_Gateway

Vaultage uses **HttpOnly Cookies** to store JWTs. This mitigates XSS attacks because the JavaScript layer cannot "read" the token; it only sends it.

// Server-Side Response (Node.js) 

res.cookie('token', jwtToken, {
              httpOnly: true,
              secure: process.env.NODE_ENV === 'production',
              sameSite: 'strict',
              maxAge: 3600000 // 1 hour
            });

02_Security_Flow

**Anti-IDOR Injection:** We never trust the `owner_id` passed from the frontend. The middleware decodes the JWT and force-injects the user's real ID into the request.

// Middleware Verification

const authMiddleware = (req, res, next) => {
              const token = req.cookies.token;
              const decoded = jwt.verify(token, process.env.SECRET);
              req.user = { id: decoded.userId }; // Secure ID Injection
              next();
            };

03_Upload_Protocol

**S3 Signed URLs:** To save server bandwidth, the server generates a temporary "Presigned URL." The browser then uploads directly to AWS S3.

// S3 Integration (SDK v3)

const command = new GetObjectCommand({
              Bucket: "vaultage-core",
